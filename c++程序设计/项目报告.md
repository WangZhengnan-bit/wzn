## 1. 项目概述

### 1.1 项目背景与目标

随着电子商务的普及，商品管理、购物车、订单处理等核心功能成为电商系统的基石。本项目旨在通过 C++ 面向对象编程技术，从零构建一个**控制台版电子商务管理系统**，模拟真实电商平台的核心业务流程。

**项目目标**：
- 深入实践面向对象三大特性：**封装、继承、多态**
- 掌握抽象基类与纯虚函数的设计方法
- 熟练运用 C++11 智能指针（`std::shared_ptr`）管理动态内存
- 实现完整的电商业务流程闭环：**用户管理 → 商品浏览 → 购物车 → 下单 → 订单管理**

### 1.2 主要功能

**用户系统**：注册、登录、管理员/顾客角色区分

**商品管理**（管理员）：
- 查看所有商品
- 添加商品（实体商品 / 数字商品）
- 删除商品、库存补货

**购物系统**（顾客）：
- 按分类浏览商品（实体/数字）
- 购物车增删改查
- 结算下单（自动扣减库存）

**订单管理**（管理员）：
- 查看所有订单、更新订单状态
- 取消订单（自动退款恢复库存）

## 2. 面向对象设计详解

### 2.1 需求分析

#### 2.1.1 项目描述

本系统是一个面向控制台的**多角色电子商务管理平台**。系统预设示例商品和用户数据，支持管理员对商品库存和订单状态进行后台管理，支持顾客浏览商品、操作购物车并完成下单。核心挑战在于：**商品类型多样**（实体/数字商品属性不同）、**用户角色不同**（管理员与顾客权限不同），需要通过合理继承结构统一管理，同时用多态简化业务逻辑。

#### 2.1.2 功能需求

| 编号 | 功能 | 角色 | 说明 |
|------|------|------|------|
| F1 | 用户注册与登录 | 所有用户 | 支持管理员/顾客两种角色注册 |
| F2 | 商品浏览 | 所有用户 | 按全量或按分类（实体/数字）浏览商品 |
| F3 | 商品添加 | 管理员 | 添加实体商品（含重量、配送区域）或数字商品（含文件大小、格式） |
| F4 | 商品删除 | 管理员 | 按商品ID删除 |
| F5 | 库存补货 | 管理员 | 按商品ID增加库存数量 |
| F6 | 加入购物车 | 顾客 | 验证库存后添加到购物车，已存在时合并数量 |
| F7 | 购物车管理 | 顾客 | 查看、修改数量、移除商品 |
| F8 | 结算下单 | 顾客 | 扣减库存，生成订单，清空购物车 |
| F9 | 订单管理 | 管理员 | 查看所有订单、更新状态、取消订单（恢复库存） |
| F10 | 订单历史 | 顾客 | 查看个人历史订单 |

#### 2.1.3 用例分析

**用例名称**：顾客完成一次购物

**参与者**：已登录的顾客

**前置条件**：顾客已成功登录，系统中存在有库存的商品

**主事件流**：
1. 顾客选择"浏览所有商品"，系统显示全部商品及库存
2. 顾客选择"添加商品到购物车"，输入商品ID和数量
3. 系统验证商品存在且库存充足，添加/更新购物车条目
4. 顾客选择"查看购物车"，确认商品和总价
5. 顾客选择"结算下单"，系统确认
6. 系统遍历购物车，逐一扣减对应商品库存
7. 系统生成订单（状态：待处理），清空购物车
8. 系统将订单记录到顾客的历史订单中

**后置条件**：订单创建成功，商品库存减少，购物车清空

#### 2.2.3 继承策略分析

**为什么选择该继承结构？**

1. **Product 继承体系**：实体商品和数字商品共享"名称、价格、库存"等基本属性，但各自有独特的附加属性（重量 vs 文件大小），且显示方式和分类标识不同。基类提取共性，派生类扩展个性，符合 **"开闭原则"**——未来新增"虚拟商品""服务类商品"只需新增派生类即可。

2. **User 继承体系**：管理员和顾客都拥有用户名、密码等基本认证信息，`checkPassword()` 逻辑完全一样，放在基类中**复用代码**；而角色标签和显示界面完全不同，通过纯虚函数 `getRole()` 和 `display()` 强制派生类各自实现，**实现多态接口**。

**protected 成员设计理由**：
- 基类 `Product` 的 `m_id`、`m_name`、`m_price`、`m_stock` 设为 `protected`：派生类 `PhysicalProduct::display()` 需调用基类 `Product::display()` 后再追加输出，派生类虽不直接访问这些成员（通过 getter），但保留 `protected` 是为了**未来可能的派生类直接操作基础属性需求**。
- 基类 `User` 的 `m_id`、`m_username`、`m_password` 设为 `protected`：`Admin::display()` 和 `Customer::display()` 直接引用 `m_id` 和 `m_username` 构造输出，这是 `protected` 的典型应用场景——**派生类访问基类成员，但不对外暴露**。

**virtual 函数设计理由**：
- `Product::display()` 声明为 `virtual` 但非纯虚：基类提供了默认的商品信息显示格式（ID/名称/价格/库存），派生类 `PhysicalProduct` 和 `DigitalProduct` 在基类显示基础上各自追加类型特有信息，体现了**多态复用**——基类指针调用 `display()` 时，运行时动态绑定到正确的派生类版本。
- `getCategory()` 和 `getRole()` 声明为**纯虚函数 `= 0`**：每个派生类必须返回不同的分类/角色标识文本，基类无法提供有意义的默认实现，纯虚函数强制派生类必须重写，同时使基类成为**抽象类**（不能直接实例化），这是正确的设计——不应存在"一个既不是实体也不是数字的 Product 对象"。

### 2.3 详细设计

#### 2.3.1 关键类的接口设计

**1. 抽象基类 `Product`**

```cpp
class Product {
protected:
    int         m_id;
    std::string m_name;
    double      m_price;
    int         m_stock;
    std::string m_description;

public:
    Product(int id, const std::string& name, double price, int stock,
            const std::string& description = "");
    virtual ~Product() = default;

    // 虚函数：基类提供默认实现，派生类可重写扩展
    virtual void        display()     const;
    // 纯虚函数：派生类必须实现，返回商品分类字符串
    virtual std::string getCategory() const = 0;

    // 公共接口（非虚函数，所有派生类行为一致）
    int  getId()          const;
    int  getStock()       const;
    bool reduceStock(int quantity);    // 扣减库存，true=成功
    void increaseStock(int quantity);  // 增加库存
};
```

**2. 派生类 `PhysicalProduct`**

```cpp
class PhysicalProduct : public Product {
private:
    double      m_weight;         // 重量（kg）
    std::string m_shippingRegion; // 配送区域

public:
    PhysicalProduct(int id, const std::string& name, double price, int stock,
                    double weight, const std::string& shippingRegion = "全国",
                    const std::string& description = "");

    void        display()     const override;  // 重写：先调基类display再追加
    std::string getCategory() const override;  // 返回 "实体商品"
};
```

**3. 核心组合类 `Store`**

#### 2.3.2 虚函数与多态的体现

以下是系统中多态的核心应用——**通过基类指针容器统一管理派生类对象，调用虚函数实现动态绑定**：

```cpp
// ========== 场景1：多态显示所有商品 ==========
void Store::displayProducts() const {
    // m_products 是 vector<shared_ptr<Product>>
    // 实际存储的是 PhysicalProduct 或 DigitalProduct 的指针
    for (const auto& product : m_products) {
        product->display();  // ★ 多态调用！
        // 运行时根据实际对象类型分发：
        //   PhysicalProduct → PhysicalProduct::display()
        //   DigitalProduct  → DigitalProduct::display()
    }
}

// ========== 场景2：按分类筛选商品 ==========
void Store::displayProductsByCategory(const std::string& category) const {
    for (const auto& product : m_products) {
        if (product->getCategory() == category) {  // ★ 多态调用纯虚函数
            product->display();                     // ★ 多态调用虚函数
        }
    }
}

// ========== 场景3：登录后根据角色分派菜单 ==========
// 在 mainMenu() 中：
auto user = store.login(username, password);
if (user->getRole() == "管理员") {   // ★ 基类指针调用纯虚函数
    adminMenu(store);               // 管理员 → 管理界面
} else {
    customerMenu(store);            // 顾客  → 购物界面
}
```

**多态带来的架构优势**：添加新商品类型时，只需新增一个 `Product` 的派生类并重写 `display()` 和 `getCategory()`，所有使用 `vector<shared_ptr<Product>>` 的代码（显示、筛选、购物车）**无需任何修改**，完美符合"对扩展开放，对修改关闭"的开闭原则。

#### 2.3.3 核心算法/业务逻辑简介

**结算下单流程**（系统中最复杂的业务流程）：

```
用户确认下单
    │
    ├─ 1. 校验购物车非空
    │
    ├─ 2. 遍历购物车商品：
    │   ├─ findProduct(id) → 获取存储中的真实商品对象
    │   ├─ product->reduceStock(qty) → 扣减库存
    │   └─ 若任一商品库存不足 → 下单失败（返回 nullptr）
    │
    ├─ 3. OrderManager.createOrder() → 创建订单（状态: Pending）
    │
    ├─ 4. ShoppingCart.clear() → 清空购物车
    │
    └─ 5. 返回订单对象 → 调用方将其加入顾客历史
```

**订单取消与库存恢复**：

```cpp
bool OrderManager::cancelOrder(int orderId) {
    auto order = findOrder(orderId);
    // 已签收或已取消的订单不可再取消
    if (order->getStatus() == OrderStatus::Delivered ||
        order->getStatus() == OrderStatus::Cancelled) {
        return false;
    }
    // 遍历订单商品，逐一恢复库存
    for (const auto& item : order->getItems()) {
        item.product->increaseStock(item.quantity);
    }
    order->setStatus(OrderStatus::Cancelled);
    return true;
}
```

## 3. 核心难点与解决方案

### 3.1 问题描述

在项目开发初期，我遇到了一个**悬垂指针（dangling pointer）导致程序崩溃**的问题。

**问题场景**：顾客将商品加入购物车时，我最初使用裸指针 `Product*` 存储商品的引用。当管理员在另一操作中删除该商品后，购物车中的指针变成了悬垂指针。之后顾客查看购物车或结算时，程序直接崩溃（访问已释放的内存）。

### 3.2 定位过程

1. **日志插桩**：在 `Store::removeProduct()` 和 `ShoppingCart::display()` 中添加 `std::cout` 输出，发现删除商品后再次访问购物车时崩溃发生在 `item.product->getName()` 调用处。

2. **GDB 调试**：
   ```bash
   gdb ecommerce.exe
   (gdb) run
   # 触发崩溃后：
   (gdb) bt          # 查看调用栈，定位崩溃点
   (gdb) print item  # 查看指针地址，发现指向的内存已被释放
   ```

3. **根因确认**：多个对象持有同一 `Product` 的裸指针，缺乏统一的生命周期管理。当 Store 删除了商品，购物车和订单中的指针全部失效。

### 3.3 解决方案

将所有涉及对象的裸指针替换为 **C++11 智能指针 `std::shared_ptr`**，利用**引用计数**机制自动管理生命周期。

**关键改动**：

```cpp
// ===== 改动前（有问题的设计）=====
class ShoppingCart {
    std::vector<std::pair<Product*, int>> m_items;  // 裸指针！
};

class Store {
    std::vector<Product*> m_products;         // 裸指针！
    // 删除商品时 delete，但购物车/订单中仍有引用 → 悬垂指针
};

// ===== 改动后（正确设计）=====
class ShoppingCart {
    // CartItem 持有 shared_ptr<Product>，引用计数+1
    std::vector<CartItem> m_items;
};

struct CartItem {
    std::shared_ptr<Product> product;  // 智能指针！
    int quantity;
};

class Store {
    // 所有容器统一使用 shared_ptr
    std::vector<std::shared_ptr<Product>> m_products;
    std::vector<std::shared_ptr<User>>   m_users;
};

// 删除商品时：从 m_products 中 erase → 引用计数-1
// 若购物车/订单中仍持有 shared_ptr → 引用计数 > 0 → 内存不被释放
// 当所有持有者都释放后 → 引用计数 = 0 → 内存自动释放
```

**效果**：彻底消除了悬垂指针问题。`shared_ptr` 的引用计数机制确保：只要还有任何一个对象（购物车、订单、商品列表）引用某商品，其内存就不会被释放。代码更加安全，且不需要手动 `delete`。

### 5.2 对面向对象设计的理解深化

通过本项目的完整开发过程，我对面向对象设计有了实质性理解：

1. **封装**：将数据和操作数据的函数绑定在一起，通过 public 接口控制访问权限。例如 `Product::reduceStock()` 内部做库存校验，外部无需关心实现细节。

2. **继承与代码复用**：基类 `Product` 中定义的属性（id、名称、价格）和方法（库存操作）被两个派生类直接复用，避免了重复代码。

3. **多态的实际价值**：`Store` 用 `vector<shared_ptr<Product>>` 一条语句管理两种商品类型，`display()` 一行调用即可正确分发到派生类版本。没有多态的话，需要用 `if-else` 或 `switch-case` 区分类型，代码将膨胀且难以扩展。

4. **抽象基类的约束力**：`getCategory() = 0` 纯虚函数强制每个派生类必须实现分类接口，从编译器层面防止"遗漏实现"的错误。

void ShoppingCart::addItem(std::shared_ptr<Product> product, int quantity) {
    if (quantity <= 0) return;

    for (auto& item : m_items) {
        if (item.product->getId() == product->getId()) {
            item.quantity += quantity;
            std::cout << "  已更新 \"" << product->getName()
                      << "\" 数量为 " << item.quantity << "\n";
            return;
        }
    }
    m_items.emplace_back(std::move(product), quantity);
    std::cout << "  已添加 \"" << m_items.back().product->getName()
              << "\" x" << quantity << " 到购物车\n";
}

void ShoppingCart::removeItem(int productId) {
    auto it = std::find_if(m_items.begin(), m_items.end(),
        [productId](const CartItem& item) {
            return item.product->getId() == productId;
        });
    if (it != m_items.end()) {
        std::cout << "  已从购物车移除 \"" << it->product->getName() << "\"\n";
        m_items.erase(it);
    } else {
        std::cout << "  购物车中未找到该商品。\n";
    }
}

void ShoppingCart::updateQuantity(int productId, int newQuantity) {
    if (newQuantity <= 0) {
        removeItem(productId);
        return;
    }
    for (auto& item : m_items) {
        if (item.product->getId() == productId) {
            item.quantity = newQuantity;
            std::cout << "  已更新 \"" << item.product->getName()
                      << "\" 数量为 " << newQuantity << "\n";
            return;
        }
    }
    std::cout << "  购物车中未找到该商品。\n";
}

void ShoppingCart::clear() {
    m_items.clear();
}

double ShoppingCart::getTotal() const {
    double total = 0.0;
    for (const auto& item : m_items) {
        total += item.product->getPrice() * item.quantity;
    }
    return total;
}

        std::cout << "  购物车为空。\n";

    std::cout << "│ 序号  商品名称            单价      数量  小计\n";
    std::cout << "├──────────────────────────────────────────┤\n";
    int index = 1;
    for (const auto& item : m_items) {
        double subtotal = item.product->getPrice() * item.quantity;

                  << std::right
                  << " ¥" << std::setw(7) << std::fixed << std::setprecision(2) << item.product->getPrice()
                  << " x" << std::setw(3) << item.quantity

    std::cout << "│ 合计: ¥" << std::fixed << std::setprecision(2) << m_total << "\n";
    std::cout << "└──────────────────────────────────────────┘\n";
}

OrderManager::OrderManager() : m_nextOrderId(10001) {}

std::shared_ptr<Order> OrderManager::createOrder(
    int customerId, const std::string& customerName,
    const std::vector<CartItem>& items)
{
    auto order = std::make_shared<Order>(m_nextOrderId++, customerId,
                                         customerName, items);
    m_orders.push_back(order);
    return order;
}

bool OrderManager::cancelOrder(int orderId) {
    auto order = findOrder(orderId);
    if (!order) return false;
    if (order->getStatus() == OrderStatus::Delivered ||
        order->getStatus() == OrderStatus::Cancelled) {
        return false;
    }
    for (const auto& item : order->getItems()) {
        item.product->increaseStock(item.quantity);
    }
    order->setStatus(OrderStatus::Cancelled);
    return true;
}

bool OrderManager::updateOrderStatus(int orderId, OrderStatus newStatus) {
    auto order = findOrder(orderId);
    if (!order) return false;
    order->setStatus(newStatus);
    return true;
}

std::shared_ptr<Order> OrderManager::findOrder(int orderId) const {
    auto it = std::find_if(m_orders.begin(), m_orders.end(),
        [orderId](const std::shared_ptr<Order>& o) {
            return o->getOrderId() == orderId;
        });
    return (it != m_orders.end()) ? *it : nullptr;
}

std::vector<std::shared_ptr<Order>> OrderManager::getOrdersByCustomer(
    int customerId) const
{
    std::vector<std::shared_ptr<Order>> result;
    for (const auto& order : m_orders) {
        if (order->getCustomerId() == customerId) {
            result.push_back(order);
        }
    }
    return result;
}

void OrderManager::displayAllOrders() const {
    if (m_orders.empty()) {
        std::cout << "  暂无订单。\n";
        return;
    }
    std::cout << "\n┌──────────────── 所有订单 ─────────────────┐\n";
    for (const auto& order : m_orders) {
        std::cout << "│ #" << order->getOrderId()
                  << " | " << order->getCustomerName()
                  << " | ¥" << std::fixed << std::setprecision(2) << order->getTotal()
                  << " | " << order->getStatusString() << "\n";
    }
    std::cout << "└────────────────────────────────────────────┘\n";
}

void OrderManager::displayOrdersByCustomer(int customerId) const {
    auto orders = getOrdersByCustomer(customerId);
    if (orders.empty()) {
        std::cout << "  该顾客暂无订单。\n";
        return;
    }
    for (const auto& order : orders) {
        order->display();
    }
}
